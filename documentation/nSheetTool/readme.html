What is node's sheet tool?

This tool is intended to make your life easier by making budgeting and accounting more straightforward. It completely automates the process of tabulating and perusing a ledger of transactions, and is also capable of generating high-quality reports on the state of your accounts.

This tool is capable of single-book accounting, ledger management, and report generation. As a result, it is intended for the following target audiences:

  • Individuals and families

  • Family businesses

  • Local artisans

  • Small organizations and clubs

  • Project teams

If you are any of these, then you might find node's sheet tool rather helpful.

How to tabulate a ledger

Any single-book ledger that has an "Account" column and a "Value" column can be tabulated. Tabulation will tell you exactly what there is in each of your accounts at any given point in time.

To tabulate, first go to the tool menu. Then press "Tabulate". The tabulator will tabulate your ledger and generate a report. You can view the report in the pop-up, or press the "Export" button to generate a Google Doc version. The Google Doc can then be saved for posterity, or shared with others as a PDF or in an email.

Data Formatting

In brief, any sheet can be handled by node's data tool so long as it meets the following requirements:

  • Named columns: All columns must have corresponding names in the first frozen row.

  • Typed columns: The second and final frozen row of any column must either specify the mathematical type corresponding to the data in the column, or be empty. This permits the data tool to ensure the quality of your data.

  • Valid data: Any sheet must have reliable, high-quality data. Erroneous data will cause the tool to produce false or misleading results.

To gaurantee data quality, node's data tool relies on typed semantic columns. This means that the column's name is what decides what the column means, not the column's number. For instance, in a ledger, the column that corresponds to the monetary value of a transaction is always the column labelled "Value". You can shuffle and rearrange the columns as you desire without impacting the tool's capabilities.

When a type mismatch is encountered, such as a date stored in a column intended for numbers, the tool will alert you immediately about the issue. It'll do so by highlighting the offending cell in red and giving you a pop-up error notification. This ensures that you won't put data in the wrong parts of a table. You can therefore use the typechecker to your advantage by having it detect data that are out of place.

The following are all the types that the typechecker recognizes:

  • The code text type, which corresponds to arbitrary text.

  • The code any type, which may contain literally anything. Note that an empty type entry is assumed to refer to the code any type.

  • The code num type, which allows only numbers. Positive numbers, negative numbers, and decimals are permissible here. However, fractions and scientific notation are currently not supported.

Examples

The tool comes with several example sheets built into it.

To deploy an example, first navigate to the tool's menu. Press the menu, then press the option that says "Examples". You can then select any example from the list.

These examples are fully-functional sheets complete with example data. You can use them to try out the tool's features and see what it can do! One may also use the examples to jump-start a new ledger or statistical analysis. Just find an example that resembles your use-case, then edit it to make it match!
