Goal

When I set out to build my first teapot 3D renderer, my goal was simply to produce a rendering of the Utah Teapot using my own code. I wasn’t aiming for a polished image; rather, I wanted to understand how rendering algorithms work by implementing one myself.

Implementation

There are many methods one can use to generate a 3D image. Most rely on linear algebra. For instance, by multiplying a point by a matrix, I could rotate it in space about some origin. Using another matrix, I could project a 3D image into a 2D plane. To simulate visual depth, I could warp points that are farther away, bringing them closer to the center of the image.

I decided to attempt rendering a 3D entity by drawing a set of points in space, rotating them using matrices, and then projecting them into 2D using a projection matrix and depth function.

I implemented the functions for rotating points and projecting them from 3D space into 2D. To make the points appear more dimensional, I shaded each as a circle with the top slightly brighter and the bottom slightly dimmer. Since points exist in 3D space, I needed to sort them before rendering. I used JavaScript’s native sorting algorithm to sort them by depth. I chose to render each point in a brass color because it looks visually pleasing. To contrast it, I set the background to a dim gray.

Finally, to test the renderer, I created a simple "always rotate" script and loaded several objects. The Utah teapot, of course, was among them. I also included the Utah teaspoon, which is less famous but still interesting. Additionally, the renderer displays a sphere and a cube.

Thoughts

All in all, this was a fun project and I'm glad I undertook it. This simple renderer helped me better understand the applications of linear algebra to practical 3D modeling. It also gave me insight into why the Utah Teapot remains so popular.

Of course, rendering only points isn't especially exciting. For my next project, I’ve decided to render the teapot using triangles instead.
